1. Основни цели
- Демонстрация на конкурентен достъп до споделени ресурси
- Работа с нишки и thread pool
- Предотвратяване на race condition чрез synchronized
- Ясна структура и разделение на отговорностите между класовете

2. Store
- Съдържа всички продукти в магазина
- Основен метод: synchronized boolean sellProduct(String productName, int amount)
- Гарантира, че само една нишка може да продава продукт в даден момент
- Проверява:
  1. дали продуктът съществува 
  2. дали има достатъчно количество

3. Product
- Представлява продукт в магазина
- Полета:
  1. name – име на продукта
  2. quantity – налично количество
- Метод: decreaseQuantity(int amount)

4. Cart
- Представлява количка на купувач
- Съдържа Map<String, Integer>:
  1. ключ → име на продукт
  2. стойност → количество

5. BuyerTask
- Имплементира Runnable
- Представлява логиката на един купувач
- При изпълнение:
  1. обхожда продуктите в количката
  2. опитва да ги закупи от магазина
  3. извежда информация за нишката и резултата

6. Използвани технологии
- Java 17+ 
- Gradle
- ExecutorService 
- Runnable
- synchronized
